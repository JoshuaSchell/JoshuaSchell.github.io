The release of C++11 in 2011 marked a turning point. After over a decade of stagnation, C++ suddenly became *modern*. C++14 and C++17 built on that foundation, giving us the language we know today. If you're still writing C++03-style code, this post is for you.

---

## C++11: The Revolution

C++11 was so transformative it was originally called "C++0x" — they expected it to ship before 2010. It didn't, but the wait was worth it.

### `auto` — Type Inference

No more typing `std::vector<std::map<std::string, int>>::iterator`. Let the compiler figure it out.

```cpp
// Before C++11
std::vector<std::string>::iterator it = names.begin();
std::map<std::string, std::vector<int>>::const_iterator cit = data.find("key");

// After C++11
auto it = names.begin();
auto cit = data.find("key");

// Works with any expression
auto x = 42;           // int
auto y = 3.14;         // double
auto z = func();       // whatever func() returns
auto& ref = container; // reference
const auto& cref = x;  // const reference
```

**Rule of thumb**: Use `auto` when the type is obvious or verbose. Be explicit when clarity matters.

### Range-Based For Loops

Iterating containers finally became intuitive.

```cpp
std::vector<int> numbers = {1, 2, 3, 4, 5};

// Before: iterator gymnastics
for (std::vector<int>::iterator it = numbers.begin(); it != numbers.end(); ++it) {
    std::cout << *it << "\n";
}

// After: just say what you mean
for (int n : numbers) {
    std::cout << n << "\n";
}

// With references (avoid copies)
for (const auto& item : heavy_objects) {
    process(item);
}

// Modifying elements
for (auto& x : numbers) {
    x *= 2;
}
```

### Lambda Expressions

Anonymous functions, inline. This changed everything for algorithms and callbacks.

```cpp
// Basic syntax: [captures](params) -> return_type { body }
auto add = [](int a, int b) { return a + b; };
std::cout << add(2, 3); // 5

// Capturing variables
int multiplier = 10;
auto scale = [multiplier](int x) { return x * multiplier; };

// Capture modes
[=]  // capture all by value
[&]  // capture all by reference
[x]  // capture x by value
[&x] // capture x by reference
[=, &y] // all by value except y by reference
[this] // capture this pointer

// Real-world: sorting with custom comparator
std::vector<std::string> words = {"apple", "pie", "a"};
std::sort(words.begin(), words.end(),
    [](const std::string& a, const std::string& b) {
        return a.length() < b.length();
    });
// Result: {"a", "pie", "apple"}

// Algorithms become expressive
std::vector<int> nums = {1, 2, 3, 4, 5, 6};
int even_count = std::count_if(nums.begin(), nums.end(),
    [](int n) { return n % 2 == 0; });
```

### Smart Pointers

Manual memory management was the #1 source of bugs. Smart pointers fix that.

```cpp
#include <memory>

// unique_ptr: sole ownership, zero overhead
std::unique_ptr<Widget> widget = std::make_unique<Widget>(); // C++14
auto w2 = std::make_unique<Widget>();
// Automatically deleted when out of scope
// Cannot be copied, only moved

// shared_ptr: shared ownership via reference counting
std::shared_ptr<Resource> r1 = std::make_shared<Resource>();
std::shared_ptr<Resource> r2 = r1; // both point to same object
// Deleted when last shared_ptr goes out of scope

// weak_ptr: non-owning observer (breaks cycles)
std::weak_ptr<Resource> weak = r1;
if (auto locked = weak.lock()) { // try to get shared_ptr
    use(locked);
}

// Factory pattern with unique_ptr
std::unique_ptr<Base> create_derived(int type) {
    switch (type) {
        case 1: return std::make_unique<DerivedA>();
        case 2: return std::make_unique<DerivedB>();
        default: return nullptr;
    }
}
```

**Guidelines:**
- Default to `unique_ptr`
- Use `shared_ptr` only when ownership is truly shared
- Use `weak_ptr` to break reference cycles
- Never use raw `new`/`delete` in modern code

### Move Semantics & Rvalue References

The performance revolution. Instead of copying expensive resources, *move* them.

```cpp
class Buffer {
    int* data_;
    size_t size_;

public:
    // Copy constructor (expensive)
    Buffer(const Buffer& other)
        : data_(new int[other.size_]), size_(other.size_) {
        std::copy(other.data_, other.data_ + size_, data_);
    }

    // Move constructor (cheap - just steal the guts)
    Buffer(Buffer&& other) noexcept
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;
        other.size_ = 0;
    }

    // Move assignment
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data_;
            data_ = other.data_;
            size_ = other.size_;
            other.data_ = nullptr;
            other.size_ = 0;
        }
        return *this;
    }
};

// std::move casts to rvalue reference
std::vector<Buffer> buffers;
Buffer b;
buffers.push_back(std::move(b)); // moves instead of copies
// b is now in "moved-from" state (valid but unspecified)
```

### `nullptr`

Type-safe null pointer, replacing the macro `NULL`.

```cpp
void process(int x);
void process(int* p);

// Before: ambiguous
process(NULL);   // calls process(int) — NULL is often 0!
process(0);      // same problem

// After: clear intent
process(nullptr); // calls process(int*)

// Works in boolean contexts
int* p = nullptr;
if (p) { /* ... */ }  // false
```

### Uniform Initialization

One syntax to rule them all.

{% raw %}
```cpp
// All of these work
int x{42};
std::vector<int> v{1, 2, 3, 4, 5};
std::map<std::string, int> m{{"a", 1}, {"b", 2}};

// Prevents narrowing conversions

// Works with custom types

// Initializer lists

    for (int v : values) std::cout << v << " ";
}
process({1, 2, 3}); // works
```

### Variadic Templates

Templates that accept any number of arguments.

```cpp
// Base case
void print() {}

// Recursive case
template<typename T, typename... Args>
void print(T first, Args... rest) {
    std::cout << first << " ";
    print(rest...); // recurse with remaining args
}

print(1, "hello", 3.14, 'x');
// Output: 1 hello 3.14 x

// sizeof... gives the count
template<typename... Args>
void info(Args... args) {
    std::cout << "Got " << sizeof...(args) << " arguments\n";
}
```

### Threading Primitives

Finally, standard threading support.

```cpp
#include <thread>
#include <mutex>
#include <future>

// Basic thread
std::thread t([]{ std::cout << "Hello from thread!\n"; });
t.join(); // wait for completion

// Mutex for synchronization
std::mutex mtx;
void safe_increment(int& counter) {
    std::lock_guard<std::mutex> lock(mtx);
    ++counter;
} // lock released automatically

// Futures for async results
std::future<int> result = std::async(std::launch::async, [](){
    std::this_thread::sleep_for(std::chrono::seconds(1));
    return 42;
});
std::cout << "Answer: " << result.get() << "\n"; // blocks until ready
```

### New Containers

```cpp
#include <array>
#include <tuple>
#include <unordered_map>
#include <unordered_set>

// std::array — fixed-size, stack-allocated
std::array<int, 5> arr = {1, 2, 3, 4, 5};
arr[0] = 10;
arr.size(); // 5, known at compile time

// std::tuple — heterogeneous fixed collection
std::tuple<int, std::string, double> t{42, "hello", 3.14};

// Hash-based containers (O(1) average lookup)
std::unordered_map<std::string, int> scores;
scores["alice"] = 95;

std::unordered_set<int> seen;
seen.insert(42);
```

## C++14: The Refinement

C++14 polished C++11's rough edges. Smaller release, but important quality-of-life improvements.

### Generic Lambdas

Lambdas with `auto` parameters — basically lambda templates.

```cpp
// C++11: must specify types
auto add11 = [](int a, int b) { return a + b; };

// C++14: let the compiler figure it out
auto add14 = [](auto a, auto b) { return a + b; };

add14(1, 2);       // int
add14(1.5, 2.5);   // double
add14(std::string("a"), std::string("b")); // string concatenation

// Useful for generic algorithms
auto print_all = [](const auto& container) {
    for (const auto& item : container) {
        std::cout << item << " ";
    }
    std::cout << "\n";
};

print_all(std::vector<int>{1, 2, 3});
print_all(std::list<std::string>{"a", "b", "c"});
```

### Return Type Deduction

Functions can deduce their return type.

```cpp
// C++14: auto return type
auto multiply(int a, int b) {
    return a * b; // deduced as int
}

// Works with complex types
auto create_map() {
    std::map<std::string, std::vector<int>> result;
    // ... populate
    return result; // type deduced
}

// Trailing return type still useful for templates
template<typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}
```

### `std::make_unique`

The missing piece from C++11.

```cpp
// C++11: awkward
std::unique_ptr<Widget> w(new Widget(arg1, arg2));

// C++14: consistent with make_shared
auto w = std::make_unique<Widget>(arg1, arg2);

// Exception-safe, no raw new
auto arr = std::make_unique<int[]>(100); // array version
```

### Relaxed `constexpr`

`constexpr` functions can now have loops, local variables, and more.

```cpp
// C++11 constexpr: single return statement only
constexpr int factorial_11(int n) {
    return n <= 1 ? 1 : n * factorial_11(n - 1);
}

// C++14 constexpr: normal function body
constexpr int factorial_14(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

constexpr int fact5 = factorial_14(5); // computed at compile time
```

## C++17: The Consolidation

C++17 brought major quality-of-life features and filled important gaps in the standard library.

### Structured Bindings

Destructure tuples, pairs, and structs directly.

{% raw %}
```cpp
// Tuples
std::tuple<int, std::string, double> get_data();
auto [id, name, score] = get_data();

// Pairs (great for map iteration)
std::map<std::string, int> scores = {{"alice", 95}, {"bob", 87}};
for (const auto& [name, score] : scores) {
    std::cout << name << ": " << score << "\n";
}

// Structs
struct Point { int x, y, z; };
Point p{1, 2, 3};
auto [x, y, z] = p;

// Arrays
int arr[] = {1, 2, 3};
auto [a, b, c] = arr;

// Even works with custom types (via get<>)
```
{% endraw %}

### `if constexpr`

Compile-time conditional compilation within functions.

```cpp
template<typename T>
auto get_value(T t) {
    if constexpr (std::is_pointer_v<T>) {
        return *t; // only compiled for pointers
    } else {
        return t;  // only compiled for non-pointers
    }
}

int x = 42;
int* p = &x;
get_value(x);  // returns 42
get_value(p);  // returns 42 (dereferenced)

// No more SFINAE gymnastics for simple cases
template<typename T>
void process(T value) {
    if constexpr (std::is_integral_v<T>) {
        std::cout << "Integer: " << value << "\n";
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << "Float: " << std::fixed << value << "\n";
    } else {
        std::cout << "Other: " << value << "\n";
    }
}
```

### `std::optional`

Maybe a value, maybe not. No more sentinel values or output parameters.

```cpp
#include <optional>

std::optional<int> find_index(const std::vector<int>& v, int target) {
    for (size_t i = 0; i < v.size(); ++i) {
        if (v[i] == target) return i;
    }
    return std::nullopt; // or just {}
}

auto result = find_index(vec, 42);
if (result) {
    std::cout << "Found at index " << *result << "\n";
} else {
    std::cout << "Not found\n";
}

// value_or for defaults
int idx = find_index(vec, 42).value_or(-1);

// Constructing with value
std::optional<std::string> name = "Alice";
std::optional<std::string> empty;
```

### `std::variant`

Type-safe union. Holds one of several types.

```cpp
#include <variant>

std::variant<int, double, std::string> v;
v = 42;         // holds int
v = 3.14;       // now holds double
v = "hello";    // now holds string

// Access with std::get (throws if wrong type)
std::string s = std::get<std::string>(v);

// Safe access with std::get_if
if (auto* p = std::get_if<int>(&v)) {
    std::cout << "It's an int: " << *p << "\n";
}

// Pattern matching with std::visit
std::visit([](auto&& arg) {
    using T = std::decay_t<decltype(arg)>;
    if constexpr (std::is_same_v<T, int>)
        std::cout << "int: " << arg << "\n";
    else if constexpr (std::is_same_v<T, double>)
        std::cout << "double: " << arg << "\n";
    else
        std::cout << "string: " << arg << "\n";
}, v);
```

### `std::string_view`

Non-owning reference to a string. Zero-copy string operations.

```cpp
#include <string_view>

// No allocation, just a pointer + length
void process(std::string_view sv) {
    std::cout << sv.substr(0, 5) << "\n";
}

std::string s = "Hello, World!";
process(s);            // no copy
process("literal");    // no copy
process(s.c_str());    // no copy

// Efficient tokenization
std::string_view text = "one,two,three";
while (!text.empty()) {
    auto comma = text.find(',');
    auto token = text.substr(0, comma);
    std::cout << token << "\n";
    text.remove_prefix(comma == std::string_view::npos ? text.size() : comma + 1);
}
```

**Warning:** The underlying string must outlive the `string_view`.

### Fold Expressions

Expand variadic templates elegantly.

```cpp
// Sum all arguments
template<typename... Args>
auto sum(Args... args) {
    return (args + ...); // unary right fold
}
sum(1, 2, 3, 4); // ((1 + 2) + 3) + 4 = 10

// Print all arguments
template<typename... Args>
void print(Args... args) {
    ((std::cout << args << " "), ...);
    std::cout << "\n";
}
print(1, "hello", 3.14); // 1 hello 3.14

// Check if all are true
template<typename... Args>
bool all_true(Args... args) {
    return (args && ...);
}

// Check if any is true
template<typename... Args>
bool any_true(Args... args) {
    return (args || ...);
}
```

### Attributes

Communicate intent to the compiler.

```cpp
// [[nodiscard]] — warn if return value is ignored
[[nodiscard]] int compute() { return 42; }
compute(); // warning: ignoring return value

// Great for error codes
[[nodiscard]] ErrorCode init_system();

// [[maybe_unused]] — suppress unused warnings
void debug_function([[maybe_unused]] int verbose_level) {
    #ifdef DEBUG
    // use verbose_level
    #endif
}

// [[fallthrough]] — intentional switch fallthrough
switch (x) {
    case 1:
        do_something();
        [[fallthrough]];
    case 2:
        do_more();
        break;
}
```

### Filesystem Library

Finally, portable file operations.

```cpp
#include <filesystem>
namespace fs = std::filesystem;

// Path manipulation
fs::path p = "/home/user/documents/file.txt";
p.filename();   // "file.txt"
p.stem();       // "file"
p.extension();  // ".txt"
p.parent_path(); // "/home/user/documents"

// Existence and type checks
if (fs::exists(p)) { /* ... */ }
if (fs::is_regular_file(p)) { /* ... */ }
if (fs::is_directory(p)) { /* ... */ }

// Directory iteration
for (const auto& entry : fs::directory_iterator("/home/user")) {
    std::cout << entry.path() << "\n";
}

// Recursive iteration
for (const auto& entry : fs::recursive_directory_iterator("/home/user")) {
    if (entry.is_regular_file()) {
        std::cout << entry.path() << " (" << entry.file_size() << " bytes)\n";
    }
}

// Create directories, copy, remove
fs::create_directories("/tmp/a/b/c");
fs::copy("src.txt", "dst.txt");
fs::remove("old_file.txt");
fs::remove_all("old_directory");
```

### Parallel Algorithms

Standard algorithms with execution policies.

```cpp
#include <execution>
#include <algorithm>
#include <numeric>

std::vector<int> data(10'000'000);

// Sequential (default)
std::sort(data.begin(), data.end());

// Parallel
std::sort(std::execution::par, data.begin(), data.end());

// Parallel + vectorized
std::sort(std::execution::par_unseq, data.begin(), data.end());

// Works with many algorithms
std::for_each(std::execution::par, data.begin(), data.end(),
    [](int& x) { x *= 2; });

auto sum = std::reduce(std::execution::par, data.begin(), data.end());

auto it = std::find(std::execution::par, data.begin(), data.end(), 42);
```

**Note:** Parallel execution requires thread-safe operations. No shared mutable state without synchronization.

## Summary: What to Use When

| Feature | When to Use |
|---------|------------|
| `auto` | Obvious types, iterators, lambdas |
| Range-for | Any container iteration |
| Lambdas | Algorithms, callbacks, local functions |
| `unique_ptr` | Default for heap allocation |
| `shared_ptr` | Only when truly sharing ownership |
| Move semantics | Returning large objects, containers |
| `optional` | "Maybe has a value" |
| `variant` | Type-safe unions, state machines |
| `string_view` | Read-only string parameters |
| `if constexpr` | Type-dependent compile-time branches |
| Structured bindings | Decomposing tuples, pairs, maps |

## Conclusion

C++11/14/17 transformed C++ from a language you *tolerated* into one you *enjoy*. The features aren't just syntactic sugar — they eliminate entire categories of bugs and make code more expressive.

If you're maintaining older codebases, start adopting these features incrementally:
1. Replace raw pointers with smart pointers
2. Use `auto` and range-for for iteration
3. Add `std::optional` for nullable returns
4. Use `string_view` for string parameters

Each change makes your code safer, more readable, and often faster.

*Next up: C++20 and C++23 push even further with concepts, ranges, and coroutines.*
